## 🛠️ 一、 已知功能性问题 (Functional Bugs)

这些是代码逻辑或技术实现上已知的小瑕疵：

### 1. 任务列表 (Task Lists) 渲染失效
*   **现象**：在 Markdown 中使用的 `- [ ]` 或 `- [x]` 语法，目前可能只会显示为普通的无序列表圆点，而不是复选框。
*   **原因**：Python-Markdown 的默认解析器未启用 Github Flavored Markdown (GFM) 的 Tasklist 扩展。
*   **临时方案**：目前请忽略文中出现的 `[ ]` 符号，后续将引入 `pymdown-extensions` 修复。

### 2. 复杂表格的兼容性
*   **现象**：虽然已实现了表格的移动端横向滚动，但如果在 `<table>` 标签中添加了自定义 CSS 类名，可能会被构建脚本的暴力替换逻辑破坏 HTML 结构。
*   **影响**：仅影响极少数需要特殊样式的表格，普通 Markdown 表格不受影响。

### 3. 标签 URL 的特殊字符处理
*   **现象**：如果标签名为 `C++` 或 `C#`，生成的 URL 可能会变成 `c.html` 或直接报错。
*   **原因**：目前的 `slugify` 逻辑比较简单，只是将空格替换为横杠，未完全转义特殊符号。

## 🎨 二、 UI/UX 阅读体验待优化

这些问题不影响功能使用，但影响“爽”的感觉：

### 1. 移动端目录 (TOC) 交互体验
*   🔴 **现状**：在手机端，点击文章顶部的“📑 文章目录”展开后，点击任意标题跳转，目录菜单不会自动收起。
*   ⚠️ **影响**：跳转后，展开的巨大目录列表依然占据屏幕，需要手动点击折叠才能看到正文。
*   ✅ **计划**：引入极少量的 Vanilla JS 来监听点击事件，实现“跳转即收起”。

### 2. 图片加载体验 (无 Lazy Load)
*   🔴 **现状**：打开一篇包含大量高清图片的长文时，浏览器会一次性加载所有图片。
*   ⚠️ **影响**：可能会消耗不必要的流量，并拖慢页面首屏渲染速度。
*   ✅ **计划**：为所有 `<img>` 标签添加 `loading="lazy"` 属性。

### 3. 缺乏“上一篇/下一篇”导航
*   🔴 **现状**：读完一篇文章滑到底部后，只有“回到顶部”链接，没有直接跳转到相邻文章的按钮。
*   ⚠️ **影响**：读者想要继续阅读时，必须跳回首页或归档页，中断了心流。

### 4. 社交媒体分享卡片 (Open Graph 缺失)
*   🔴 **现状**：当您将文章链接分享到微信、Twitter 或 Telegram 时，只能显示标题和链接，无法显示文章的封面图或摘要卡片。
*   🔍 **原因**：缺少 HTML `<head>` 中的 `og:image` 和 `twitter:card` 元数据。

## 🚀 三、 未来改进路线图 (Roadmap)

为了贯彻极简与高性能的理念，以下功能已被列入开发计划：

### [ P0 - 近期修复 ]
- [ ] **修复 Favicon**：目前浏览器标签页缺少网站图标。
- [ ] **增强无障碍性 (A11y)**：增加按钮和链接的 `aria-label` 属性，优化键盘导航焦点的可见性。
- [ ] **深色模式闪烁问题**：优化 CSS 载入顺序，防止页面刷新时出现短暂的“白屏闪烁”。

### [ P1 - 中期规划 ]
- [ ] **站内搜索功能**：引入一个纯前端的搜索方案（如 `lunr.js` 或简单的 JSON 索引匹配），在不依赖后端数据库的情况下实现秒级搜索。
- [ ] **相关文章推荐**：基于标签 (Tag) 的重合度，在文章底部自动生成“你可能也感兴趣”列表。
- [ ] **代码块一键复制**：在代码高亮块的右上角添加“复制”按钮。

### [ P2 - 探索中 ]
- [ ] **RSS 订阅优化**：现在的 RSS 仅包含摘要，未来考虑输出全文 RSS，方便阅读器用户。
- [ ] **WebP 图片自动转换**：构建时自动将 jpg/png 转换为 webp 格式，进一步压缩体积。
