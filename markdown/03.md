为了持续提升用户体验并解决已发现的样式和维护问题，特列出以下九项待处理事项：

## 🛠️ UI/UX 待解决问题

1.  **移动端标题栏显示异常：** 在部分移动设备上，顶部标题栏（Header）存在显示 Bug，影响初次加载体验。
2.  **深色模式样式适配不足：** 标题栏及导航栏在深色模式（Dark Mode）下的背景和文字颜色适配不完善，与页面主体风格不一致。
3.  **移动导航栏过于拥挤：** 在窄屏设备上，顶部导航链接元素间距不足，显示效果拥挤。
4.  **页脚内容优化：** 需要进一步优化底部页脚（Footer）的显示内容和排版，使其更简洁或更具信息量。
5.  **Emoji 字符显示异常：** 页面上的 Emoji 表情符号无法正常显示，需要检查字体或字符集配置。
6.  **代码块主题待更换：** 浅色模式下，代码显示区域（Code Block）的颜色方案（高亮主题）不够美观，计划更换为更舒适的主题。
7.  **取消首字下沉效果：** 不希望文章段落的第一个字出现放大的首字下沉（Drop Cap）效果，需从 CSS 中移除。
8.  **顶部导航栏圆角适配：** 顶部导航栏的直角与页面中其他卡片（如文章内容区）的圆角设计不统一，应进行圆角处理以适配整体设计风格。

## ⚙️ 维护及数据问题

9.  **部署时间更新异常：** 文章页面底部的部署时间（Build Time）有时不能根据 Markdown 文件的修改时间（`mtime`）自动更新，需要检查 Python 生成器逻辑或 GitHub Actions 提交机制。
