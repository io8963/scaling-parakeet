## ✨ 核心功能与优势

以下是本站目前已实现的功能及其为读者带来的好处：

### 1. 极致的速度与性能

| 功能 | 效果与好处 |
| :--- | :--- |
| **纯静态生成** | 页面在访问前已完全生成，服务器无需处理请求，**页面加载速度极快**，近乎瞬时。 |
| **增量构建** | **（网站管理员）** 网站内容更新速度加快，文章发布更加及时。 |
| **CSS 哈希缓存** | CSS 文件名包含哈希值（`style.xxxxxx.css`），确保您浏览器**永远加载最新的样式**，同时旧样式在更新后能被快速替换。 |
| **图片懒加载 (Lazy Load)** | 仅当图片进入浏览器视口时才开始加载。显著减少了**页面首次加载所需的数据量**和**渲染时间**。 |

### 2. 优秀的内容呈现与交互

| 功能 | 效果与好处 |
| :--- | :--- |
| **完整文章目录 (TOC)** | 每篇文章都提供**锚点链接**的目录，方便您快速跳转到感兴趣的章节。 |
| **表格包裹器** | 自动为所有表格添加容器 (`<div class="table-wrapper">`)，确保表格在**小屏幕上可以水平滑动**，避免内容溢出屏幕。 |
| **代码高亮** | 使用 Pygments 进行**服务端渲染**的高亮代码块，支持多种语言，提供清晰的阅读体验。 |
| **跨篇导航** | 在文章末尾提供**上一篇（较新）** 和 **下一篇（较旧）** 文章链接，提升阅读连贯性。 |

### 3. 可访问性与用户体验

| 功能 | 效果与好处 |
| :--- | :--- |
| **暗色模式（Dark Mode）** | 网站支持通过 CSS **自动适应**您操作系统的暗色模式设置（`prefers-color-scheme: dark`）。 |
| **跳过链接 (Skip Link)** | 对于使用键盘或屏幕阅读器的用户，可以快速跳过导航直接进入**主内容区域**。 |
| **构建时间显示** | 每篇文章和主页底部都精确显示了该页面**最近一次的构建时间**（基于 Git Commit 记录），方便您了解信息的时效性。 |

## 🛑 目前存在的局限性

在坚持**不使用 JavaScript** 的前提下，我们必须接受一些功能上的取舍。这是目前网站的主要局限性：

1.  **缺少站内搜索功能：** 无法提供实时、便捷的站内搜索框。读者需要依赖**外部搜索引擎**（如 Google、Baidu）进行关键词搜索，并在关键词前加上 `site:yourdomain.com`。
2.  **没有用户自定义暗色模式切换：** 网站的亮色/暗色模式完全由**操作系统/浏览器偏好**决定。用户无法手动点击按钮进行切换。
3.  **缺少评论系统：** 无法在页面上直接集成动态评论功能。

## 🎯 后续改进方向

我们承诺继续在纯静态技术的限制下，不断探索和改进阅读体验。未来的优化将集中在以下几个方面：

### 1. 进一步提升性能

* **增量构建增强：** 优化构建清单的追踪细节，确保即使是最小的元数据（如标题、日期）变更，也能最小化地重新构建受影响的页面，进一步加快网站更新速度。

### 2. 优化阅读体验

* **图片响应式优化：** 探索在构建时自动生成多尺寸图片 (`srcset`) 的可能性，让浏览器在不同设备上加载最合适的图片，进一步减少带宽消耗。
* **元数据结构化：** 引入 **JSON-LD** 结构化数据，帮助搜索引擎更好地理解文章内容（例如作者、发布日期、文章类型），有助于在搜索结果中显示**更丰富的摘要信息**。

### 3. 增强可访问性

* **ARIA 角色完善：** 仔细检查并为所有导航、主内容区添加适当的 ARIA 属性和角色，以最大程度地优化屏幕阅读器的体验。

我们相信，一个专注、快速且无需担忧隐私泄露的阅读环境，才是数字内容最好的载体。感谢您的支持！
